Deployment

Vercel 适合部署静态站吗

官方地址：vercel.com

Vercel 很适合部署静态站、前端应用和轻量全栈项目。把 GitHub 仓库导入后，每次推送都会自动部署，适合个人项目快速上线，也适合团队用预览链接检查改动。

适合做什么

  • 部署纯静态站、工具导航、文档站和作品集。
  • 通过 GitHub 推送触发自动构建和发布。
  • 绑定自定义域名，使用预览部署检查每次改动。

使用建议

纯静态项目可以选择 Other 框架，构建命令留空，输出目录使用项目根目录。上线后要检查正式域名、404 页面、robots.txt、sitemap.xml、ads.txt 和移动端布局。

注意事项

如果页面依赖客户端读取 JSON，最好用 HTTP 服务预览，不要只双击 HTML 文件。绑定域名后要等待 DNS 和证书生效，期间访问异常不一定是代码问题。

推荐搭配

GitHub 负责代码和提交记录，Vercel 负责部署，Search Console 负责提交 sitemap 和观察收录情况。
