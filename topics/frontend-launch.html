Launch Checklist

前端静态站上线前检查清单

适合：Vercel 静态站、工具导航、个人作品集、内容站。

静态站上线看起来简单，但容易漏掉移动端、SEO、错误页和缓存策略。上线前按清单走一遍，能减少发布后才发现标题不对、页面打不开、资源缓存异常等问题。

上线前先查三类资料

  • Web API 和语义标签：优先查 MDN，避免使用过时写法。
  • 浏览器兼容性：使用 Can I use 确认关键 CSS 和 JS 特性。
  • 社区问题：遇到部署报错时再查 Stack Overflow 或 GitHub Issues。

静态站必须检查的文件

首页需要清晰的 title 和 description，内容页也应该有独立标题。robots.txt 用来告诉搜索引擎哪些路径可以抓取，sitemap.xml 用来列出主要页面。404 页面要能从错误路径返回首页或主要内容页，避免用户撞到空白页。

布局和性能检查

至少检查桌面、平板和手机宽度。重点看按钮文字是否溢出、卡片是否挤压、导航是否遮挡内容。图片、图标和 JSON 数据可以设置较短缓存，频繁更新的内容不要缓存太久。对于工具导航站，搜索框和分类列表应该比装饰视觉更优先。

发布顺序

先本地预览，再推送 GitHub，最后看 Vercel Deployments。部署成功后打开正式域名检查首页、内容页、404、ads.txt、robots.txt 和 sitemap.xml。
