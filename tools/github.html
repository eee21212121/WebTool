Code Hosting

GitHub 适合个人项目和团队协作吗

官方地址：github.com

GitHub 不只是代码仓库。它还能管理 Issue、Review、Actions 自动化和项目发布。对于个人项目，它能保存历史和部署状态；对于团队，它能建立协作流程和质量门槛。

适合做什么

  • 托管代码、文档和静态站项目。
  • 通过 Issue 和 Pull Request 记录需求、问题和改动过程。
  • 使用 GitHub Actions 自动运行测试、构建或部署任务。

使用建议

个人项目也建议保持清晰提交信息。团队项目则要约定分支策略、Review 规则和发布流程。README 应该说明项目用途、本地运行方式和部署方式，否则过一段时间自己也会忘记。

注意事项

不要把密钥、数据库密码、私有配置和个人隐私提交到仓库。即使后来删除，历史记录里也可能残留。公开仓库尤其要检查 `.env`、构建产物和截图内容。

推荐搭配

GitHub 管理代码，Vercel 负责部署，MDN 和 Stack Overflow 负责查文档与排错。
